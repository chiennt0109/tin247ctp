⚡ Độ phức tạp thuật toán: đọc Big-O và ước lượng thời gian chạy

Bài học này giúp học sinh hiểu vì sao cùng một bài toán nhưng có lời giải chạy rất nhanh, có lời giải lại bị quá thời gian. Trọng tâm là đọc được O(1), O(log n), O(n), O(n log n), O(n²) và biết chọn thuật toán phù hợp với giới hạn đề.

Mục tiêu bài học

Học xong cần làm được

  • Hiểu khái niệm độ phức tạp thời gian của thuật toán.
  • Biết cách đọc, ước lượng và so sánh hiệu quả giữa các giải pháp.
  • Phân biệt O(1), O(log n), O(n), O(n log n), O(n²).
  • Nhìn vào vòng lặp để đoán nhanh độ phức tạp.

Ví dụ đời sống

  • Tìm một quyển sách bằng cách xem từng quyển: gần giống O(n).
  • Tìm từ trong từ điển bằng cách mở giữa rồi chia đôi: gần giống O(log n).
  • So sánh từng cặp học sinh trong lớp: gần giống O(n²).

1. Độ phức tạp thuật toán là gì?

Độ phức tạp thuật toán mô tả tốc độ tăng của số phép tính khi đầu vào lớn dần. Ta không đếm chính xác từng phép cộng, phép gán, phép so sánh, mà quan tâm xu hướng tăng theo n.

Cách hiểu đơn giản: Nếu n tăng gấp 10 lần, số việc thuật toán phải làm tăng khoảng bao nhiêu lần? Câu trả lời này quan trọng hơn việc máy tính chạy nhanh hay chậm ở một test nhỏ.
O(1)

Hằng số. Ví dụ truy cập a[i].

O(log n)

Chia đôi liên tục. Ví dụ Binary Search.

O(n)

Duyệt một vòng qua mảng.

O(n²)

Hai vòng lặp lồng nhau.

2. So sánh tốc độ tăng

Dưới đây là mô phỏng số phép tính tương đối khi thay đổi n. Hãy thử nhập n = 10, 100, 1000, 100000 để thấy sự khác biệt.

Lưu ý: Các số dưới đây là ước lượng để học sinh cảm nhận tốc độ tăng, không phải thời gian chạy chính xác.

3. Minh họa bằng code: O(n) và O(1)

Cùng tính tổng từ 1 đến n, ta có hai cách. Cách dùng vòng lặp là O(n); cách dùng công thức là O(1). 

#include <bits/stdc++.h>
using namespace std;

int main() {
    long long n;
    cin >> n;

    long long s = 0;
    for (long long i = 1; i <= n; i++) {
        s += i;                 // chạy n lần
    }

    cout << s;
    return 0;
}
#include <bits/stdc++.h>
using namespace std;

int main() {
    long long n;
    cin >> n;

    long long s = n * (n + 1) / 2;   // tính trực tiếp

    cout << s;
    return 0;
}

5. Ước lượng nhanh khi làm bài thi

Khi đọc đề, hãy nhìn giới hạn n trước. Nếu n lớn, thuật toán O(n²) thường không phù hợp.

Kết quả ước lượng sẽ hiện ở đây.
Mốc tham khảo phổ biến: khoảng 10⁷ đến 10⁸ phép tính/giây trong C++ tùy máy và loại phép tính. Đây chỉ là kinh nghiệm ước lượng, không phải quy tắc tuyệt đối.

6. Mô phỏng: O(n) khác O(n²) như thế nào?

O(n): một vòng lặp

Với n = 10, thuật toán làm khoảng 10 việc.

O(n²): hai vòng lặp

Với n = 10, thuật toán làm khoảng 100 cặp việc.

Nhấn “Bước tiếp” để xem số bước tăng như thế nào.

7. Lỗi học sinh hay nhầm

Nhầm tốc độ máy với độ phức tạp.
Máy nhanh chỉ giúp chạy nhanh hơn một chút; thuật toán sai độ phức tạp vẫn có thể TLE khi n lớn.
Thấy 2 vòng lặp là luôn kết luận O(n²).
Nếu hai vòng lặp tách rời, có thể là O(n + n) = O(n). Nếu lồng nhau, thường mới là O(n²).
Không quan tâm giới hạn đề bài.
Trước khi code, cần xem n tối đa để chọn thuật toán.
Giữ cả hằng số và hệ số quá chi tiết.
O(3n + 5) thường viết gọn thành O(n).

8. Bài tập luyện tập phân loại

📘 Mức 1 - Nhận biết

  1. Xác định độ phức tạp của đoạn code có một vòng lặp từ 1 đến n.
  2. Xác định độ phức tạp của đoạn code truy cập a[5].
  3. Giải thích vì sao O(2n) thường viết là O(n).

📗 Mức 2 - So sánh

  1. Viết hàm tính tổng từ 1 đến n theo cách tuyến tính O(n).
  2. Viết công thức trực tiếp tính tổng O(1).
  3. So sánh hai cách với n = 10⁶.

📙 Mức 3 - Phân tích code

  1. Cho đoạn code hai vòng lặp lồng nhau, xác định độ phức tạp.
  2. Phân tích thuật toán tìm kiếm nhị phân có độ phức tạp O(log n).
  3. Giải thích vì sao sắp xếp thư viện thường là O(n log n).

🐉 Mức 4 - Định hướng HSG

  1. Với n = 2 · 10⁵, đề yêu cầu xử lý nhiều truy vấn, hãy loại bỏ các hướng O(nq).
  2. Cho bài tìm cặp tổng bằng K, so sánh O(n²), O(n log n), O(n).
  3. Đọc giới hạn đề và dự đoán nhóm thuật toán phù hợp.
💡 Xem lời giải mẫu Python so sánh O(n) và O(1) 
import time

def sum_linear(n):
    s = 0
    for i in range(1, n + 1):
        s += i
    return s

def sum_formula(n):
    return n * (n + 1) // 2

n = 1000000

start = time.time()
sum_linear(n)
print("O(n):", round(time.time() - start, 6), "s")

start = time.time()
sum_formula(n)
print("O(1):", round(time.time() - start, 6), "s")
#include <bits/stdc++.h>
using namespace std;

long long sum_linear(long long n) {
    long long s = 0;
    for (long long i = 1; i <= n; i++) s += i;
    return s;
}

long long sum_formula(long long n) {
    return n * (n + 1) / 2;
}

int main() {
    long long n = 1000000;
    cout << sum_linear(n) << "\\n";
    cout << sum_formula(n) << "\\n";
    return 0;
}
