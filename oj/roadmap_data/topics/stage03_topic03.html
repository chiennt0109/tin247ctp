🎯 Mục tiêu bài học

  • Hiểu khái niệm độ phức tạp thời gian của thuật toán.
  • Biết cách đọc, ước lượng và so sánh hiệu quả giữa các giải pháp.
  • Phân biệt O(1), O(n), O(n log n), O(n²).

📘 Khái niệm cơ bản

Độ phức tạp thuật toán mô tả tốc độ tăng của số phép tính khi đầu vào lớn dần.

Ví dụ: với n phần tử:

  • O(1): Hằng số – truy cập phần tử thứ i trong mảng.
  • O(n): Tuyến tính – duyệt toàn bộ mảng.
  • O(n log n): Hiệu quả – ví dụ Merge Sort, Quick Sort.
  • O(n²): Chậm hơn – ví dụ Bubble Sort, Insertion Sort.

📈 Minh họa bằng code

C++

#include <bits/stdc++.h>
using namespace std;
int main() {
    int n; cin >> n;
    long long s = 0;
    for (int i = 1; i <= n; i++) s += i; // O(n)
    cout << s;
}

Python

n = int(input())
s = sum(range(1, n+1))  # O(n)
print(s)

🧩 Bài tập luyện tập

  1. Viết hàm tính tổng từ 1 đến n theo cách tuyến tính (O(n)).
  2. Viết công thức trực tiếp tính tổng (O(1)).
  3. So sánh thời gian chạy của hai cách trên với n = 10⁶.
💡 Xem lời giải mẫu 
# Python
import time

def sum_linear(n):
    s = 0
    for i in range(1, n + 1):
        s += i
    return s

def sum_formula(n):
    return n * (n + 1) // 2

n = 1000000
start = time.time()
sum_linear(n)
print("O(n):", round(time.time() - start, 6), "s")

start = time.time()
sum_formula(n)
print("O(1):", round(time.time() - start, 6), "s")
